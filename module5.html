Processing Arguments, Putting it All Together

In this module, we will look at two topics in processing arguments. The first is processing varialble numbers of arguments to varidic functions, and the facilities in C that enable this. We will also learn how to pass arguments to a program and how to use values passed to costumize program behavior. We will examine several libraries that are available that provide a convenient way to consistently process arguemnts and option values specified by users when the program is run.

With the advent of social media on the web, a new phenomenon known as a chatbot" has becoming more pervasive. A chatbot is a computer program that conducts a conversation via text or voice. Such programs are often designed to convincingly simulate how a human would behave as a conversational partner, thereby passing the Turing test. In this final module, we will briefly survey the history of chatbot like programs, and examine a simple chatbot that uses many of the techniques we have studied in this course.

Processing Arguments

So far, we have only used functions with a fixed number of arguments. In C and many other languages, it is possible to create functions with variable numbers of arguments. We have already seen an example of a function with a variable number of arguments: printf(), which takes a format string and additional arguments that provide values to the format string. We will look at how to process variable numbers of arguments and several examples of how this technique is used.

A programs can also be passed arguments that can be used to provide data, and to customize the program behavior . Values can be passed when running an application from a command line like “bash” in Unix, or “cmd.com” or “powershell” in Windows., or from within an IDE such as Eclipse. We will look at several ways to process and use program arguments.

A Chatbot Named "ELIZA"

According to Joseph Weizenbaum, he wrote ELIZA between 1964 and 1966 as a parody of “the responses of a non-directional [Rogerian] psychotherapist in an initial psychiatric interview.”. The program examined the text input by a user for keywords, and transformed the input into an output. The DOCTOR script determined the keywords, set the values of keywords, and set the rules of transformation for the output.

The program gave deceptively intelligent responses that deceived many individuals when first using the program. In fact, ELIZA’s responses were so convincing that Weizenbaum and several others have anecdotes of users becoming emotionally attached to the program, occasionally forgetting that they were conversing with a computer.

Weizenbaum’s own secretary reportedly asked Weizenbaum to leave the room so that she and ELIZA could have a real conversation. Weizenbaum was surprised by this, later writing, “I had not realized… that extremely short exposures to a relatively simple computer program could induce powerful delusional thinking in quite normal people.”

Chatbots are now big business, with applications from influencing social media, to taking over from human telephone solicitors (apparently very successfully among among certain demographic groups). There are even companies who specialize in creating custom chatbots for companies to deploy

Additional Exercises

Here are additional exercises that will give you more practice with the subject material in this module.

Variadic Functions

  1. Create a function double avg(unsigned n, ...) that computes the average of its double variadic arguments. The number of arguments is specified by n. If the number of arguments is 0, return the double constant NaN.
  2. Create a function unsigned maxDistance(int val, unsigned n, ...) that takes a value and a number of int variadic arguments specified by n. The function computes the furthest absolute distance from the value passed in to any of the variadic arguments. If n is 0, return 0.
  3. Create a function unsigned wordCount(const char*, ...) that takes one or more const char* parameters to strings. The parameter list is terminated by a NULL pointer. Assume that a word is delimited by a single space (' ') or by the end of string character ('\0'), as with assignment 3. The function returns the number of words in the parameter strings.
  4. Extend the vaPrintf() function shown in lecture 14 to interpret a non-standard format specifier "%[d" to print an array of integers. The format specifier expects the next argument to be an unsigned array count, and the argument after that to be a pointer to an array of that number of ints. The specifier prints the array as a bracketed list, whose fields are separated by commas (','). For example, vaPrintf("%[d", 3, array_of_3) with the values for array_of_3 as 2, 4, and 6 would print "[2,4,6]".

Program Arguments

For these problems, use the form of main function that accepts an argument count argc and an array of argument strings argv: int main(int argc, int argv[argc]). The argument count is always at least 1, and the first argument (argv[0]) is always the name of the program.
  1. Create a C program whose main function prints its name on the first line, and then additional argument strings, one per line, as "argv[%d]: %s\n" and returns with EXIT_SUCCESS. Test this with 0, 2, and 4 additional program arguments.
  2. Create a C program whose main function looks for an optional first short argument that can be either "-l" or "-u". If first argument is the option "-l" the program prints lower-cased versions of its additional program arguments one per line, as "argv[%d]: %s\n". If the first argument is the option "-u' the program prints upper-cased versions of its additional arguments. If the first argument is any other option beginning with '-' the program prints a usage message and returns EXIT_FAILURE. Otherwise, the program prints its additional program arguments in their original forms and returns with EXIT_SUCCESS.
  3. Create a version of the previous C program to accept an optional first long argument that can be either "--lower" or "--upper".
  4. Modify the short option verions of the previous C program to accept an additional optional short program argument "-n count" that specifies the number of times to repeat each additional argument on its output line, separated by spaces. This option can be specified either before or after the "-l" or "-u" option if they are present. The count is required if the "-n" option is specified and the program returns EXIT_FAILURE if it is not specified.
  5. Modify the long option version of the previous C program to accept an additional long argument "--nreps=count" and the program returns EXIT_FAILURE if the count is not specified.
  6. Use the standard C getopt argument processing functions to implement the previous short option C program.
  7. Use the standard C getopt argument processing functions to implement the previous long and short option C programs as a single program.
