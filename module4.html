Memory management, Modularization, and Unit Testing

We begin this module by learning about how memory is managed in a C program, and about a new capability for dynamically creating and managing data in a portion of memory that is independent of fixed global storage and the local storage within a given function.

Next we will learn about how to modularize program functionality into multiple files. Often, each module contains the data definitions and funcitons for a single data type, with the main function and associated logic in their own files.

Finally, we will also look at the concept of unit testing and at strategies for unit testing a program, and learn how to use the CUnit unit test framework to imlement unit tests that are included in either their own module or in the same moudule as the main function.

Memory Management

In this lecture, we will learn about how memory is managed in a C program, including the stack, the constant pool, and the heap. We will also learn about functions that enable us to use the C heap for dynamically creating and managing data in a this portion of memory that is independent of fixed global storage and the local storage within a given function.

Modularizing C Programs Into Separate Compilation Units

In this lecture, we will learn about how to modularize program functionality into multiple files to make code easier to create and manage. In many cases, each file contain the data definitions and funcitons for a single data type, with the main function and associated logic in their own files.

We will also learn about the concept of sharing declarations for the the functions in a given file with functions in other files using external storage declarations, and about how to use the C preprocessor to put related declarations for data types and external function declarations into their own files and include them in files that require them.

Unit Testing

Testing the data defintions and code we create is an extremely important part of software development. As we create functionality and modify it over time, designing and creating good unit tests enables us to ensure that the code we write performs correctly, both as it is being developed and when it is modified over time.

Until now, we have included simple calls to functions as a way to verify they work correctly. Now we will learn about the strategies of designing unit tests, and use the CUnit unit test framework to imlement unit tests that are included in either their the main function module or in their own module.

Additional Exercises

Here are additional exercises that will give you more practice with the subject material in this module.

  1. Write a function createPoint() that takes two double paramaters x and y, and returns a pointer to a Point that is dynamically allocated and initialized by the function. Assume the definition of the typedef Point struct we have used in class and in earlier exercises.
  2. Write a function createCircle() that takes a pointer to a center Point and a double radius. The function returns a pointer to a Circle that is dymanically allocated and initiailized by the function. The function copies the information from the input Point to the embedded Point in the Circle. Assume the definition of the typedef Circle struct we have used in class and in earlier exercises.
  3. Write a function createRectangle() that takes a pointer to the upper left Point, a double width and height. The function returns a pointer to a Rectangle that is dymanically allocated and initiailized by the function. The function copies the information from the input Point to the embedded Point in the Rectangle. Assume the definition of the typedef Rectangle struct we have used in class and in earlier exercises.
  4. Write a function getBoundingBox() that takes a pointer to a Circle. The function returns a pointer to a Rectangle that is set to the bounding box for for the Circle. Use the createRectangle() function from the previous problem to create and initialize the Rectangle.

    5. The previous four problems would be similar to a final exam question, with the Point, Circle, and Rectangle struct typedefs provided.

  5. Write a function repString() that takes a pointer to a string and an unsigned int count. The function creates and returns a pointer to a string that is dynamically allocated and initialized to n repetitions of the input string.

    6. The previous problem would be similar to a final exam question

  6. Write a function splitString() that splits a string into a list of strings at the delimiter character ch. The function dynamically allocates storage for a copy of the input string, and replaces each occurence of ch in the copy by the null character ('\0'). It also sets the character after the final null character in the copy to the character '\377' (octal), to mark the end of the list. The function returns a pointer to the string list copy.

    7. The previous problem would be similar to a final exam question. The following two functions would be provided for testing purposes.

  7. Write a function nextSplitString() that, given a pointer to a split string list, returns a pointer to the beginning of the next string in the list, or NULL if there is no next string. The start of the next string in the list is the current pointer plus the string length plus 1.
  8. Write a function splitStringSize() that, given a pointer to a split string list, returns the number of strings in the list including the current one as a size_t.
